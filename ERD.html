Schema Overview

The system uses a MySQL database with the following core entities:

  • Inventory: products
  • Transactions: sales, sale_items, payments
  • CRM: customers (Senior Citizens / PWDs)
  • Security & Auditing: users, audit_logs, void_logs
  • Configuration: settings

Table Descriptions

Inventory

  • products: Stores all inventory items. cost_price (Buying Price) and price (Selling Price) allow for profit calculation. stock_quantity is updated automatically upon sale completion.

Sales & Transactions

  • sales: The header table for transactions. Stores total_amount, status, and discount details. discount_id logically links to the customers.id_number.
  • sale_items: Detailed line items for each sale. It snapshots unit_price, original_cost, and product_name at the time of sale. This preserves historical profit margin data even if product costs change later.
  • payments: Records payment details such as amount tendered and change given.

Customer Management (CRM)

  • customers: Stores persistent data for Senior Citizens (SC) and Persons with Disability (PWD). Used for faster checkout by retrieving ID numbers and addresses across sessions.

Auditing & Security

  • audit_logs: A generic system-wide audit trail for critical actions (e.g., login, inventory updates).
  • void_logs: Specific, immutable logs for voided sales, ensuring security and accountability for cancelled transactions.
  • users: Stores system users with role-based access control (RBAC).

System Configuration

  • settings: Key-value store for system configurations (e.g., store name, receipt headers, backup paths).
